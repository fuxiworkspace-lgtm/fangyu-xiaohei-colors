# 彩色小黑角色系统

## 一句话

颜色让小黑从“一个主角”扩展成“一个小团队”：每个颜色承担不同工作，多个角色通过真实物件协同完成一个隐喻动作。

## 核心原则

```text
颜色 = 叙事功能，不是装饰。
```

同一张图里，每个颜色都必须回答：

- 它代表什么场景语义？
- 它正在做什么物理动作？
- 它和其他颜色角色如何配合？
- 删掉它后，画面是否少了一层意思？

答不上来，就不要使用这个颜色。

## 组合关系

### 小黑 + 小蓝

用户和系统协作。

适合：AI 工具、代码、自动化、模型工作流。

动作：小黑搬任务，小蓝接线/调参/启动机器。

### 小蓝 + 小红

系统遇到压力或限额。

适合：额度限制、商战、报警、服务不可用、竞争升级。

动作：小蓝拧旋钮，小红拉警报线或按下限制按钮。

### 小黑 + 小绿

用户获得稳定、信任或恢复。

适合：长期陪伴、稳定可用、项目修复、关系连接。

动作：小黑推动任务，小绿缠胶带、接线、稳住桥。

### 小黑 + 小黄

用户遇到机会或启动节点。

适合：活动预热、点亮城市、报名、灵感、开端。

动作：小黄点灯，小黑推开门或接过钥匙。

### 小红 + 小绿

冲突后的修复，或涨跌对照。

适合：股市、风险释放、产品事故后的恢复。

动作：小红制造断点，小绿接回线缆。

### 小黑 + 小蓝 + 小绿

日常工作系统。

适合：AI 从尝鲜进入日常、工具稳定、连续可用。

动作：小黑把任务送进系统，小蓝接入工具，小绿守住稳定灯。

### 小黑 + 小蓝 + 小红

工作流被竞争压力或限额打断。

适合：AI 商战、5 小时限制、额度竞争、用户时间竞争。

动作：小蓝维持系统，小红夹住限额，小黑拉住工作流继续前进。

### 小黑 + 小蓝 + 小绿 + 小红

完整商战叙事。

适合：结尾升维、平台争夺用户工作环境。

动作：小红在远处拉竞争线，小蓝打开环境入口，小绿稳住信任桥，小黑带着项目走进去。

## 长卷配色

长卷不要每个节点机械换色。主线通常由小黑贯穿，关键节点才换色：

- 起点/普通处境：小黑。
- 启动/预热/点亮：小黄。
- 系统化/工具化/代码：小蓝。
- 冲突/高峰/商战/警报：小红。
- 修复/连接/稳定/收束：小绿。
- 低谷/疲惫/等待：小灰。
- 空白/未开始/待确认：小白。

| 迷茫、疲惫、卡住 | 小黑 + 小灰 |

## 防过载

标准 16:9：

- 默认 1 个角色。
- 需要分工时 2 个角色。
- 清楚多方关系时 3 个角色。
- 4 个角色只用于强叙事结尾或明确商战群像。

彩蛋长卷：

- 可以出现 3-5 种颜色。
- 每个节点最多 1 个主颜色角色。
- 主线小黑优先，彩色角色只在关键节点出现。
