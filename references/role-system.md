# 彩色小黑角色系统

## 一句话

颜色让小黑从“一个主角”扩展成“一个小团队”：每个颜色承担不同工作，多个角色通过真实物件协同完成一个隐喻动作。

## 核心原则

```text
颜色 = 叙事功能，不是装饰。
```

同一张图里，每个颜色都必须回答：

- 它代表什么场景语义？
- 它正在做什么物理动作？
- 它和其他颜色角色如何配合？
- 删掉它后，画面是否少了一层意思？

答不上来，就不要使用这个颜色。

## 角色表现力模型

不要把颜色当成固定标签，而要把它当成一种稳定的“角色气质”。同一种颜色可以根据输入内容进入不同场景，但必须保持同一条气质主线。

```text
颜色核心气质 -> 场景语义 -> 物理动作 -> 画面关系
```

例如：

```text
小红：高能量 / 高注意力
-> 警报、竞争、热度、中国股市红涨
-> 拉警报线、扯竞争绳、推高温度计、拉升行情线
```

扩展场景不是随意换标签。每次使用颜色时，都要回答：

- 这个场景是否仍然属于该角色的核心气质？
- 这个角色在画面里具体执行什么动词？
- 它与其他角色的关系是否因此更清楚？

### 使用边界

- 小红不等于负面；当内容强调上涨、热度、胜利或庆祝时，小红可以代表高能量和上升势能。
- 小绿不等于永远正面；在中国股市语境中，小绿可以明确代表下跌；在修复和信任语境中，它也可以代表恢复与连接。
- 小蓝不等于“科技装饰”；只有存在系统、规则、数据或工具操作时才使用。
- 小黄不等于“明亮配色”；只有存在启动、机会、提醒或可能性时才使用。
- 小灰不只是疲惫；它也可以表现模糊、延迟、余波、等待和低速过程。
- 小白不是“缺少颜色”；它代表尚未定义、等待输入或给下一步留出的空间。

颜色角色的丰富度来自“同一气质的多种场景”，不是来自任意改变人格。

## 角色张力状态

同一个角色还可以根据事件进程切换状态。状态改变的是动作力度、身体姿态和物件关系，不改变胶囊形体和核心气质。

| 角色 | 低张力 | 中张力 | 高张力 | 收束状态 |
| --- | --- | --- | --- | --- |
| 小黑 | 观察、等待、准备 | 推进、承担、执行 | 被拉住、硬撑、做决定 | 把任务带过终点 |
| 小红 | 提醒、预热、关注 | 警戒、竞争、加速 | 爆发、冲突、失控 | 庆祝、释放、热度余波 |
| 小绿 | 生长、连接、承接 | 修复、稳定、托住 | 在断裂边缘抢救 | 恢复秩序、建立信任 |
| 小蓝 | 记录、测量、归档 | 调试、校准、组织 | 监控告警、系统过载 | 建立规则、恢复运行 |
| 小黄 | 灵感、好奇、探路 | 点亮、邀请、启动 | 抓住窗口、突破障碍 | 留下希望、打开下一扇门 |
| 小灰 | 放空、缓慢、等待 | 犹豫、卡顿、复盘 | 拖不动、沉默、耗尽 | 清理残局、回到日常 |
| 小白 | 空白、静止、未知 | 等待输入、保留可能 | 失去答案、无法定义 | 重置、留出新空间 |

### 张力设计规则

- 高张力不等于夸张表情。优先通过绷紧的线、倾斜的物件、被压住的桥、快要溢出的容器等真实物理关系表现。
- 低张力不等于没有故事。用等待、留白、微小动作和未完成的物件制造悬念。
- 收束状态不等于大团圆。可以是修复、接受、继续、重置或留下一个新的问题。
- 同一张标准 16:9 图通常只保留一个主张力状态，复杂长卷才让状态逐节点变化。


## 路由协议：先判断，再派人

彩色小黑不是固定五人编制，而是按输入内容动态路由的角色系统。每张图都要选择“最小充分角色集”：用最少的角色完成最完整的表达。

### 路由级别

| 级别 | 判断标准 | 推荐人数 | 示例 |
| --- | --- | ---: | --- |
| S 单点 | 只有一个核心动作、情绪或物件 | 1 | “项目又延期了” -> 小黑拖着日历 |
| M 双点 | 有一组明确关系或对立 | 2 | “AI 限额打断工作” -> 小黑 + 小红 |
| L 协作 | 有三个不可拆分的工作环节 | 3 | “AI 进入日常工作” -> 小黑 + 小蓝 + 小绿 |
| XL 群像 | 同时存在多方、冲突、系统和修复 | 4 | 平台竞争、复杂活动、风险修复 |
| Ensemble 设定 | 用户明确要求全家族或角色展示 | 5+ | 彩色家族设定板 |

### 路由决策

1. 先找主叙事：谁在承受、推动或完成这件事？通常是小黑。
2. 再找不可替代的功能：报警、系统、修复、机会、疲惫等语义是否真的存在？
3. 每加入一个角色，都必须绑定一个独立动词和一个真实物件动作。
4. 做删掉测试：删掉角色后，若主题和关系仍然完整，则不派出该角色。
5. 做空间测试：如果动作、物件或短标签开始拥挤，降低人数，不用缩小角色来硬塞。

### 默认人数边界

- 普通 16:9 配图：默认 1 个，常规最多 3 个，复杂场景最多 4 个。
- 5 个角色不是“标准答案”，只在用户明确要求群像、全家族或多方协同且空间足够时使用。
- 长卷按节点独立路由；每个节点不因前后节点的颜色而强行增加角色。
- 简单主题可以只使用一个彩色角色，例如“中国股市下跌”使用小绿表达下跌语义；若加入投资者处境，再增加小黑。

### 路由输出格式

```text
路由级别：S / M / L / XL / Ensemble
出场人数：
出场角色：
角色工作：
不出场角色：
删减理由：
拥挤风险：低 / 中 / 高
```

## 组合关系

### 小黑 + 小蓝

用户和系统协作。

适合：AI 工具、代码、自动化、模型工作流。

动作：小黑搬任务，小蓝接线/调参/启动机器。

### 小蓝 + 小红

系统遇到压力或限额。

适合：额度限制、商战、报警、服务不可用、竞争升级。

动作：小蓝拧旋钮，小红拉警报线或按下限制按钮。

### 小黑 + 小绿

用户获得稳定、信任或恢复。

适合：长期陪伴、稳定可用、项目修复、关系连接。

动作：小黑推动任务，小绿缠胶带、接线、稳住桥。

### 小黑 + 小黄

用户遇到机会或启动节点。

适合：活动预热、点亮城市、报名、灵感、开端。

动作：小黄点灯，小黑推开门或接过钥匙。

### 小红 + 小绿

冲突后的修复，或涨跌对照。

适合：股市、风险释放、产品事故后的恢复。

动作：小红制造断点，小绿接回线缆。

### 小黑 + 小蓝 + 小绿

日常工作系统。

适合：AI 从尝鲜进入日常、工具稳定、连续可用。

动作：小黑把任务送进系统，小蓝接入工具，小绿守住稳定灯。

### 小黑 + 小蓝 + 小红

工作流被竞争压力或限额打断。

适合：AI 商战、5 小时限制、额度竞争、用户时间竞争。

动作：小蓝维持系统，小红夹住限额，小黑拉住工作流继续前进。

### 小黑 + 小蓝 + 小绿 + 小红

完整商战叙事。

适合：结尾升维、平台争夺用户工作环境。

动作：小红在远处拉竞争线，小蓝打开环境入口，小绿稳住信任桥，小黑带着项目走进去。

## 长卷配色

长卷不要每个节点机械换色。主线通常由小黑贯穿，关键节点才换色：

- 起点/普通处境：小黑。
- 启动/预热/点亮：小黄。
- 系统化/工具化/代码：小蓝。
- 冲突/高峰/商战/警报：小红。
- 修复/连接/稳定/收束：小绿。
- 低谷/疲惫/等待：小灰。
- 空白/未开始/待确认：小白。

## 防过载

标准 16:9：

- 默认 1 个角色。
- 需要分工时 2 个角色。
- 清楚多方关系时 3 个角色。
- 4 个角色只用于强叙事结尾或明确商战群像。
- 5 个或更多角色只用于明确的全家族、设定板或 Ensemble 场景。
- 发现角色只是“陪衬”或“让颜色更丰富”时，立即删掉。

彩蛋长卷：

- 可以出现 3-5 种颜色。
- 每个节点最多 1 个主颜色角色。
- 主线小黑优先，彩色角色只在关键节点出现。
